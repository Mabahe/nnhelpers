Content-Elemente rendern

Ein Inhaltselement rendern. Wird data übergeben, können im Inhaltselement auch Fluid-Variablen ersetzt werden.

{nnt3:contentElement(uid:123, data:feUser.data)}

Fluid-Template/HTML-Code rendern. Die Variablen in data werden im Fluid-Template ersetzt. 

{data.bodytext->nnt3:contentElement(data:'{greeting:\'Hallo!\'}')}

Spalte (colPos) aus dem Backend-Layout für eine Seite rendern.

{nnt3:content.column(colPos:110)}
{nnt3:content.column(colPos:110, pid:99)}

Rohes tt_content-Data-Array einer Spalte (colPos) aus dem Backend-Layout holen. Per default werden auch die Relationen
(FAL, assets, media...) geladen. Kann per relations:0 verhindert werden.

{nnt3:content.columnData(colPos:110)}
{nnt3:content.columnData(colPos:110, pid:99, relations:0)}
Page-Data und Seitentitel

Seiten-Titel (den <title>-Tag) ändern. 

{nnt3:page.title(title:'Seitentitel')}

Aktuelle Page-UID (pid) holen. Eigene Felder müssen in ext_localconf.php als rootLine definiert werden! 

{nnt3:page.data(key:'uid')}

Wert aus aktuellem page-data holen, mit slide. Eigene Felder müssen in ext_localconf.php als rootLine definiert werden!
Kann per \nn\t3::Registry()->rootLineFields(['key', '...']); gemacht werden. 

{nnt3:page.data(key:'media', slide:1)}
{nnt3:page.data(key:'nnp_contact', slide:1)}
{nnt3:page.data(key:'backend_layout_next_level', slide:1, override:'backend_layout')}
